# 交付物 5：AI 工具使用记录（使用总结 · 500 字内）

在本题包开发中，主要使用 Cursor 类对话式 AI 辅助完成工作。典型环节包括：快速生成与 `api-spec.md` 对齐的 TypeScript 类型与 `fetch` 封装；在实现 AI 侧栏时，对照题目要求梳理「chat → tool_calls → 调业务 API → 再 chat」的循环结构，由 AI 起草初版后再人工核对边界（空 `content`、未知工具名、错误提示）。在集成 Ant Design 与自定义样式混用时，用 AI 检索常见布局写法以减少查文档时间。

效果上，样板代码与接口拼接效率较高；但在 **React Hooks 依赖、状态不可变更新、Vitest 与 `tsc` 同时类型检查** 等细节上，AI 容易给出「能跑但不严谨」的写法，需要人工对照 ESLint/TS 报错逐项修正。例如误用 `useState` 初始化函数做副作用、或测试里 mock 掉整个 `useEffect` 导致断言失真，这类问题必须靠开发者对框架语义做最终判断。

整体上，AI 适合加速搭建与查错线索，不适合替代对需求与运行时行为的验收；本提交中的交付文档与测试场景也经人工裁剪，保证与真实代码行为一致。（请按你实际使用习惯修改本段事例与比例。）
